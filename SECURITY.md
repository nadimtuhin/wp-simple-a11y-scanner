# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Please report security issues by emailing **nadimtuhin@gmail.com** with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

You will receive a response within **72 hours**. We aim to release a fix within **14 days** of confirmation.

We appreciate responsible disclosure and will credit reporters in the release notes (unless you prefer to remain anonymous).
